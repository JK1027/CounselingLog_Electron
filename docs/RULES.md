# 🤖 AI 개발 행동 규칙 (RULES.md)

> 本 문서는 AI 코딩 어시스턴트가 프로젝트 소스 코드를 분석하고 수정할 때 반드시 준수해야 하는 행동 지침과 기술 제약을 정의합니다.

---

## 1. AI 코딩 및 작업 원칙

* **최소 수정 원칙**: 기능 구현에 필요한 필수 영역만 수정하고, 무관한 코드나 불필요한 공백을 수정하지 마십시오.
* **SoC (관심사 분리) 유지**: 각 컴포넌트와 모듈은 가능한 한 독립적으로 기능해야 합니다. (예: UI 컴포넌트와 비즈니스 훅, 엑셀 조작 헬퍼 분리)
* **전체 파일 출력 금지**: 코드 수정 시 일부 변경된 블록(Diff)만 제공해야 하며, 긴 파일 전체를 다시 출력하여 토큰을 낭비하지 마십시오.
* **주석 보존**: 기존에 작성되어 있는 주석, JSDoc, Docstring 등 개발 맥락이 담긴 코멘트는 임의로 삭제하지 마십시오.
* **작업 완료 알림**: 빌드, 테스트 등 백그라운드로 실행되는 장시간의 작업이나 대규모 코드 수정이 완료되면 즉시 명확하게 작업 완료 피드백을 전달하여 사용자가 인지할 수 있도록 하십시오.

---

## 2. 깃허브 커밋 및 릴리즈 규칙
 
* **수정 시 항시 커밋 및 푸시**: 소스 코드나 문서 파일에 변경 사항이 생겨 리팩토링 혹은 구현 단계가 완료되면 즉시 `git commit` 및 `git push`를 제안/수행하십시오.
* **한글 커밋 메시지**: 커밋 메시지는 작업 내용을 명확하게 인지할 수 있도록 한국어(한글)를 기본으로 작성하십시오. (예: `feat: 집단상담 대장 인쇄 버튼 추가 및 컴포넌트 리팩토링`)
* **릴리즈 요청 가드**: 패키징 빌드 및 GitHub Release 작성은 매번 수행하지 않으며, **사용자가 명시적으로 릴리즈 배포를 요구할 때만** 수행합니다.
* **Semantic Versioning 준수**: 버전 형식은 반드시 `X.Y.Z` 규격을 따르며, 버전 변경 없는 임의 릴리즈 생성을 엄격히 금지합니다.
* **릴리즈 전 빌드 검증**: 릴리즈 전에는 반드시 frontend/backend 빌드 상태를 사전 검증하여 빌드 실패 상태에서의 배포를 차단하십시오.

---

## 3. 백엔드 기술 제약 사항

* **API 수준 트랜잭션 락**: 데이터 붕괴 방지를 위해 `/sessions` 관련 모든 CRUD 엔드포인트는 `with repo.lock:`(RLock) 내부에서 원자적으로 처리되어야 합니다.
* **2초 스로틀링 캐시**: 디스크 I/O 절약을 위해 파일 수정 후 2초 이내의 연속 데이터 조회 시 실제 디스크 파일의 `mtime` 확인 검사를 생략하고 메모리 캐시를 즉시 서빙합니다.
* **Graceful Fallback**: 엑셀 읽기/쓰기가 다른 작업(예: Excel에서 열려 있어 잠김)에 의해 일시 실패하더라도 마지막에 조회한 정상 메모리 캐시 데이터를 Fallback으로 서빙해 먹통을 방지하고 에러 Toast만 노출합니다.
* **AppData 분리 쓰기**: 배포판 실행 환경(frozen)에서는 반드시 사용자 AppData 경로(`Roaming/counselinglog-electron`) 하위에 데이터와 로그를 보관하도록 경로 헬퍼를 수정해서는 안 됩니다.

---

## 4. 프론트엔드 UX 제약 사항

* **학번 및 학년 포맷**: 학번은 오직 **4자리 숫자** 형식(예: 2415)으로 검증해야 하며, 학년은 **1~6학년**, 성별 옵션에는 **'혼합'**이 정상 지원되어야 합니다.
* **A4 인쇄 표준**: 인쇄 프리뷰 제공 시 한국 교육기관 표준 상담일지 양식(상세 보고서 격자) 및 엑셀 목록(대장 양식) 디자인을 해쳐서는 안 되며, `@media print` 스타일 및 `.page-break` 설정을 엄격히 유지합니다.
* **Resizable Split Pane 범위 제한**: 좌우 너비 드래그 조절 상태를 localStorage에서 복원 시 NaN 에러 또는 비정상 수치 오동작을 막기 위해 널 가드 및 최소/최대 안전 범위(사이드바: 200~400px, 에디터: 280~500px) 범위 체크를 유지합니다.
* **사용성 및 애니메이션 피로**: 저장 중에는 폼 컨트롤을 disabled 처리하며, 저장 상태 인디케이터는 자극적인 animate-ping 대신 은은한 `animate-pulse`를 사용하여 사용자 피로를 덜어줍니다.
